# [Project] ZONBER : Survive the Zone

## 1. 개요 (Identity)
* **프로젝트 명:** ZONBER (존버)
* **장르:** 하이퍼 캐주얼 / 탄막 생존 (Bullet Hell Survival)
* **플랫폼:** Android / iOS (Mobile)
* **개발 스택:** Flutter (Language: Dart) + Flame Engine
* **백엔드:** Firebase (Firestore, Cloud Functions)

## 2. 핵심 컨셉 (Concept)
* **"Zone에서 버텨라":** 좁은 구역에 갇혀 쏟아지는 탄막을 피하는 극한의 생존 게임.
* **비주얼 테마:** Retro Neon & Dark Space.
    * 배경: 칠흑 같은 어둠 (#0B0C10).
    * 오브젝트: 형광색(Neon) 라인 아트.
* **플레이 경험:**
    * **압도적 물량:** 화면을 뒤덮는 아주 작은(Tiny) 탄막.
    * **광활한 맵:** 카메라가 플레이어를 추적(Follow)하며 넓은 공간감을 제공.
    * **긴장감:** 탄막 접근 시 화면 테두리 경고(Warning) 표시.

## 3. 게임 시스템 (Gameplay)
### A. 기본 규칙
* **승리 조건:** 없음 (Time Attack). 0.01초라도 더 오래 버티는 것이 목표.
* **패배 조건:** 플레이어 히트박스에 탄막이 스치는 즉시 Game Over.
* **조작:** 터치 드래그 (1:1 반응) 또는 가상 조이스틱 (설정 가능).

### B. 맵 시스템
1. **공식 맵 (Official):**
    * 난이도별 3종 (Classic / Reflect / Homing).
2. **커스텀 맵 (UGC):**
    * 유저가 직접 제작하고 공유하는 맵.
    * **[제작 3대 원칙]:**
        1. **개방성:** 외곽 테두리의 30% 이상 개방 필수.
        2. **활동성:** 맵의 50% 이상 빈 공간 유지 (스폰 구역 벽 설치 불가).
        3. **검증:** 제작자가 직접 30초 이상 생존해야 업로드 가능.

### C. 랭킹 시스템 (Ranking)
* **아케이드 스타일:** 로그인 절차 최소화.
* **기록:** 게임 오버 시 닉네임(이니셜 3~8자) 입력 후 랭킹 등록.
* **구조:** 맵(Map)마다 별도의 리더보드 존재.

## 4. UI/UX 및 기능
* **메인 로비:** 오락실 기계 느낌, [START] 버튼 강조.
* **에디터:** 20x20 그리드 타일 방식, 터치로 벽 설치/삭제.
* **설정 (Settings):**
    * 조작 민감도(Sensitivity) 조절.
    * **터치 오프셋(Touch Offset):** 손가락이 캐릭터를 가리지 않게 Y축 거리 조절.
    * BGM/SFX 볼륨, 왼손잡이 모드.

## 5. 데이터베이스 설계 (Firestore)
* **`users`**: 유저 통계, 보유 스킨, 즐겨찾기 맵.
* **`maps`**: 맵 메타정보, 그리드 데이터(`grid_data`), 제작자 검증 여부.
    * **Sub-collection `records`**: 해당 맵의 유저별 최고 기록(랭킹 데이터).

## 6. 개발 로드맵 (Roadmap)
* **Phase 1 (MVP):** 프로젝트 세팅, 플레이어 이동, 무한 탄막 생성(오브젝트 풀링), 충돌 처리.
* **Phase 2 (Data):** Firebase 연동, 랭킹 시스템(입/출력) 구현.
* **Phase 3 (Editor):** 맵 에디터 UI 및 유효성 검사 로직 구현.
* **Phase 4 (Polish):** 사운드, 그래픽(잔상 효과), 광고 연동 및 출시.
